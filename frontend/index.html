About the Project

Hardware Setup

Our IoT smart energy meter prototype featuring ESP32 microcontroller, OLED display, voltage and current sensors connected via breadboard.

How It Works

  • ESP32 with isolated voltage and current sensors (e.g., ACS712 + voltage sensor module)
  • Measures real-time RMS voltage, current, and computes power
  • Logs data for each circuit/appliance
  • Data is sent to this web dashboard over Wi-Fi

Project Objectives

  • Reduce energy costs
  • Detect phantom loads
  • Improve safety via overcurrent/overvoltage alerts
  • Provide clear, user-friendly visualizations
  • Enable data-driven energy management decisions

User Stories

The Smart Energy Meter was designed around three main users: the homeowner, the installation technician, and the power company.

Homeowner

Daily power overview: As a homeowner, I want to open the app and quickly see how much electricity I used today (for example as a simple bar chart), so I can compare it with previous days.

High-use alerts: As a homeowner, I want a notification if my house suddenly starts using a lot more power than usual, so I can turn off unnecessary appliances and avoid a high bill.

  • The dashboard shows a clear bar graph of today's usage.
  • The graph refreshes automatically roughly once every hour.

Technician

Quick QR setup: As a technician, I want to set up the smart meter just by scanning a QR code on the device with my phone, instead of typing in long IDs or configuration codes.

  • Each meter has a scannable QR code printed on the side.
  • After scanning, the basic setup completes in under 2 minutes.

Power Company

Automatic monthly readings: As a power company, I want the system to automatically send a summary of how much power each home used at the end of the month, so I can generate accurate bills without manual meter reading.

  • The system can generate a monthly usage report for a customer.
  • A report is sent automatically (for example by email) on the 1st of each month.
